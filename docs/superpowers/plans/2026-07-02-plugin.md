# Tech Debt Tracker — Claude Code Plugin Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Package the skill + a bundled CLI as an installable Claude Code plugin, with a marketplace manifest in this repo, so a teammate installs everything with `/plugin marketplace add <repo>` + `/plugin install techdebt` — no PATH setup, no clone, no yarn.

**Architecture:** New `plugin/` workspace containing `.claude-plugin/plugin.json` and `skills/techdebt/` (the skill plus an esbuild-bundled single-file CLI at `skills/techdebt/bin/techdebt.cjs`, committed like the action bundle — the existing CI drift guard covers it automatically). The plugin's SKILL.md becomes the SINGLE source of the skill (the old `skill/` directory is removed); it invokes the bundled CLI via `${CLAUDE_SKILL_DIR}` (confirmed available in skill markdown per code.claude.com/docs/en/skills.md). Repo-root `.claude-plugin/marketplace.json` makes this repo itself the marketplace. `plugin.json` deliberately omits `version`: git-SHA versioning means every commit is an update — right for team dogfooding (code.claude.com/docs/en/plugins-reference.md#version-management).

**Tech Stack:** unchanged; esbuild (already a devDep pattern from `action/`).

### Task 1: Plugin workspace + bundled CLI + manifests

**Files:**
- Create: `plugin/package.json`, `plugin/.claude-plugin/plugin.json`, root `.claude-plugin/marketplace.json`
- Modify: root `package.json` (workspaces)
- Generated (committed): `plugin/skills/techdebt/bin/techdebt.cjs`, `yarn.lock`

**Interfaces:** after this task, `yarn build` produces the plugin bundle topologically (core → cli → action → plugin), and `node plugin/skills/techdebt/bin/techdebt.cjs --help` prints the CLI help.

- [ ] **Step 1: Plugin package**

Create `plugin/package.json`:

```json
{
  "name": "@techdebt/plugin",
  "version": "0.0.0",
  "private": true,
  "type": "module",
  "scripts": {
    "build": "esbuild ../packages/cli/src/index.ts --bundle --platform=node --target=node20 --format=cjs --outfile=skills/techdebt/bin/techdebt.cjs"
  },
  "dependencies": {
    "@techdebt/cli": "workspace:*"
  },
  "devDependencies": {
    "esbuild": "^0.25.0"
  }
}
```

(The `@techdebt/cli` dependency exists purely so `yarn workspaces foreach -At` orders the bundle after the CLI build; esbuild resolves `@techdebt/core` through its built `dist` like the action bundle does.)

- [ ] **Step 2: Plugin manifest**

Create `plugin/.claude-plugin/plugin.json`:

```json
{
  "name": "techdebt",
  "description": "Tech-debt ledger for your repo: Claude finds debt and proposes estimates, you confirm in chat, simple arithmetic ranks it. The techdebt CLI is bundled — nothing else to install.",
  "author": {
    "name": "Mike McEwen"
  }
}
```

(`version` is omitted on purpose — git-SHA versioning, every commit is an update.)

- [ ] **Step 3: Marketplace manifest**

Create root `.claude-plugin/marketplace.json`:

- [ ] **Step 4: Wire the workspace and build**

Root `package.json`: `"workspaces": ["packages/*", "action", "plugin"]`.
