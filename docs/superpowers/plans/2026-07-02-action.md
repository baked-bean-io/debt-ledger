> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build build-order step 4 — the GitHub Action (`action/`) per DESIGN.md Q8 — plus the accepted follow-ups from the previous two final reviews (NaN-safe CLI option parsing, location type-check, shared todayIso, forward-slash path normalization, KEY_ORDER exhaustiveness guard).

**Architecture:** A small JS action (`node20` runtime) that bundles `@techdebt/core` via esbuild into a committed `action/dist/index.js`. Pure logic (comment body, match scoring) lives in `action/src/comment.ts` and is unit-tested; `main.ts` is a thin octokit shell (same thin-shell precedent as the CLI's interactive triage). Surface-only v1: no LLM, path-match changed files against ledger locations, one sticky comment upserted via a hidden marker, silent on clean PRs, ledger read from the PR head checkout.

**Tech Stack:** unchanged plus `@actions/core`, `@actions/github`, `esbuild` (action package only — core stays zero-dep).

## Global Constraints

- All prior global constraints still bind (ESM `.js` imports in TS source, canonical writes, commit trailer `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`, tests/commands from repo root).
- Action behavior (DESIGN.md Q8, verbatim): one sticky comment identified by `<!-- techdebt-tracker -->`; rewritten each run; **edited, not deleted** when matches drop to zero; **no comment ever created for a clean PR**; ledger read from the PR head (the checkout); body shows the same fields as `report` (id, score, status, effort/impact, title, matched files); recognizably a view over the ledger, not a second opinion.
- Comment table ordering = the ledger's deterministic ranking (core `rank` over the matched items) — never a new ordering.
- The bundle `action/dist/index.js` is COMMITTED (GitHub Actions convention); `.gitignore` must ignore `packages/*/dist/` but not `action/dist/`.
- esbuild bundle: `--bundle --platform=node --target=node20 --format=cjs` (the runner executes plain `node dist/index.js`; CJS avoids ESM-resolution issues in the bundle).
- `action/` joins the yarn workspaces; root `yarn build` must build it topologically after core.
- Errors in the action → `core.setFailed(message)`, never an unhandled rejection.

Create `action/src/comment.ts`:

```ts
import { rank, type AdjacencyMatch, type DebtItem } from '@techdebt/core';

export const MARKER = '<!-- techdebt-tracker -->';

export interface ScoredMatch {
  item: DebtItem;
  score: number;
  files: string[];
}

// Ordering comes from the ledger's deterministic ranking — the comment is a
// view over the ledger, never a second opinion (DESIGN.md Q8).
export function scoreMatches(matches: AdjacencyMatch[]): ScoredMatch[] {
  const filesById = new Map(matches.map((m) => [m.item.id, m.files]));
  return rank(matches.map((m) => m.item)).map((r) => ({
    item: r.item,
    score: r.score,
    files: filesById.get(r.item.id) ?? [],
  }));
}

function escapeCell(text: string): string {
  return text.replace(/\|/g, '\\|');
}

export function buildCommentBody(matches: ScoredMatch[]): string {
  if (matches.length === 0) {
    return `${MARKER}\nThis PR no longer touches any tracked tech debt.`;
  }
  const lines = [
    MARKER,
    '### Tech debt in files this PR touches',
    '',
    '| id | score | status | e/i | title | matched files |',
    '| --- | --- | --- | --- | --- | --- |',
  ];
  for (const m of matches) {
    lines.push(
      `| ${m.item.id} | ${m.score.toFixed(2)} | ${m.item.status} | ${m.item.effort}/${m.item.impact} | ${escapeCell(m.item.title)} | ${m.files.map((f) => `\`${f}\``).join(', ')} |`,
    );
  }
  lines.push('', '_Ordered by the ledger\'s deterministic score. Run `techdebt report` locally for the full list._');
  return lines.join('\n');
}
```

- [ ] **Step 1: Implement main.ts**

Create `action/src/main.ts`:

```ts
import { existsSync, readFileSync } from 'node:fs';
import { join } from 'node:path';
import * as core from '@actions/core';
import * as github from '@actions/github';
import { LEDGER_PATH, matchChangedFiles, parseLedger, SCHEMA_VERSION, type Ledger } from '@techdebt/core';
import { buildCommentBody, MARKER, scoreMatches } from './comment.js';

async function run(): Promise<void> {
  const pr = github.context.payload.pull_request;
  if (!pr) {
    core.info('Not a pull_request event; nothing to do.');
    return;
  }

  const token = core.getInput('github-token', { required: true });
  const octokit = github.getOctokit(token);
  const { owner, repo } = github.context.repo;

  // The ledger is read from the checkout, i.e. the PR head — a PR that fixes
  // an item and flips it to `fixed` isn't nagged about it (DESIGN.md Q8).
  const ledgerPath = join(process.env.GITHUB_WORKSPACE ?? '.', LEDGER_PATH);
  const ledger: Ledger = existsSync(ledgerPath)
    ? parseLedger(readFileSync(ledgerPath, 'utf8'))
    : { version: SCHEMA_VERSION, items: [] };

  const files = await octokit.paginate(octokit.rest.pulls.listFiles, {
    owner,
    repo,
    pull_number: pr.number,
    per_page: 100,
  });
  const changed = files.map((f) => f.filename);
  const scored = scoreMatches(matchChangedFiles(changed, ledger.items));

  const comments = await octokit.paginate(octokit.rest.issues.listComments, {
    owner,
    repo,
    issue_number: pr.number,
    per_page: 100,
  });
  const existing = comments.find((c) => c.body?.includes(MARKER));

  if (scored.length === 0 && !existing) {
    core.info('No tracked debt touched; staying silent.');
    return;
  }

  const body = buildCommentBody(scored);
  if (existing) {
    await octokit.rest.issues.updateComment({ owner, repo, comment_id: existing.id, body });
    core.info(`Updated sticky comment (${scored.length} match(es)).`);
  } else {
    await octokit.rest.issues.createComment({ owner, repo, issue_number: pr.number, body });
    core.info(`Created sticky comment (${scored.length} match(es)).`);
  }
}

run().catch((error: unknown) => {
  core.setFailed(error instanceof Error ? error.message : String(error));
});
```
